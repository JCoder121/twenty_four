# Twenty-Four тАФ Design Spec

Date: 2026-07-13
Status: approved pending user review

## Overview

A terminal-based deterministic simulation of the 24 game. The program draws 4 cards
at a time (no replacement) from 1 or 2 standard 52-card decks (no jokers) and prints
an arithmetic expression that makes 24 from the 4 card values, or `NO RESULT` if none
exists. Allowed operations: `+ - * /` and parentheses. No exponents, no roots.

This is game logic only. A future ASCII-art / frontend layer (in the style of
`eighty_points_claude`) is explicitly out of scope but should not be precluded тАФ
hence pure, I/O-free core modules.

## Decisions (locked)

- **Stack:** TypeScript / Node, zero runtime dependencies. Run via `npx tsx src/index.ts`
  (`npm start`). Tests via vitest.
- **Location:** `~/Documents/claude_playground/twenty_four/` (new project; the old
  `cursor_projects/sample_projects/game_24` stub is unrelated and untouched).
- **Card values:** A=1, 2тАУ10 face value, J=11, Q=12, K=13. All 52 cards playable.
- **Solutions shown:** one canonical expression plus a distinct-solution count,
  e.g. `(13 - 9) * (11 - 5) = 24` / `3 distinct solutions`. Unsolvable тЖТ `NO RESULT`.
- **Deck exhaustion:** prompt `Deck exhausted. Reshuffle? (y/n)`. `y` rebuilds the
  full deck(s) and continues; `n` prints the session summary and exits.
- **Solver:** recursive reducer (see below).
- **Precomputed table:** `data/solutions.json` covering all 1,820 value multisets,
  generated by `npm run precompute`, checked into the repo as a reference. Loaded at
  startup for O(1) lookup; live solve is the fallback if the file is absent.

## Components

### rational.ts

Exact rational numbers so division never introduces float error (`8/(3-8/3)` must
equal exactly 24).

- `Rational = { n: number; d: number }`, always normalized (gcd reduced, d > 0).
- `add, sub, mul, div` (div by zero returns `null`), `equals`, `fromInt`, `toString`.
- Numerators/denominators stay tiny for 4-card expressions; `number` is safe (no bigint).

### cards.ts

- `Rank = 'A' | '2' тАж '10' | 'J' | 'Q' | 'K'`, `Suit = 'тЩа' | 'тЩе' | 'тЩж' | 'тЩг'`.
- `Card = { rank: Rank; suit: Suit }`.
- `valueOf(card): number` тЖТ 1тАУ13.
- `formatCard(card)` тЖТ `"AтЩа"`, `formatHand(cards)` тЖТ `"9тЩг 5тЩе JтЩа KтЩж"`.

### deck.ts

- `buildDecks(count: 1 | 2): Card[]` тЖТ 52 or 104 cards.
- `shuffle(cards, rng)` тАФ FisherтАУYates, injectable RNG.
- `Deck` class or closure with `draw(4): Card[] | null` (null when < 4 remain) and
  `remaining(): number`. No replacement: drawn cards leave the deck.

### solver.ts

- `solve(values: number[]): { expression: string; count: number } | null`.
- Recursive reducer: from the list of rationals, pick every unordered pair, combine
  with `+`, `*`, both orders of `-` and `/` (skipping division by zero), recurse on
  the reduced list. Base case: one value; success if it equals 24 exactly.
- Every success carries its expression tree. Trees are canonicalized to count
  distinct solutions:
  - `+`/`-` chains flatten to a multiset of positive terms and a multiset of
    subtracted terms, each canonicalized recursively, sorted.
  - `*`/`/` chains flatten likewise into numerator and denominator factor multisets.
  - Distinct = distinct canonical forms. So `a+b` vs `b+a` and `(a+b)+c` vs `a+(b+c)`
    are one solution; genuinely different constructions count separately.
- Suits are irrelevant to distinctness; duplicate values (e.g. two 5s in a hand) are
  interchangeable тАФ solutions differing only by swapping equal values count once.
- `expression` is the canonical rendering of one solution, fully parenthesized only
  where precedence requires, e.g. `(13 - 9) * (11 - 5)`.

### table.ts + scripts/precompute.ts

- Multiset key: values sorted ascending, comma-joined тАФ `"1,5,5,13"`.
- `precompute.ts` iterates all C(16,4)=1,820 non-decreasing 4-tuples over 1тАУ13,
  runs `solve`, writes pretty-printed JSON:
  `{ "1,1,1,1": null, "1,1,1,2": null, тАж, "3,3,8,8": { "expression": "8 / (3 - 8 / 3)", "count": 1 }, тАж }`.
- `table.ts`: `lookup(values)` тАФ key into the loaded JSON; if the file is missing or
  the key absent, call `solve` directly. The game never breaks without the file.

### game.ts / index.ts

Prompt flow (all input validated; invalid input reprompts):

```

тАж

Continue? (y/n)
тАж
```

Session summary on any exit after тЙе1 hand:
`Hands played: N | Solvable: X | NO RESULT: Y`.

`index.ts` is the only file touching `process.stdin/stdout`; `game.ts` receives
`{ prompt(question): Promise<string>, print(line): void }`.

## Error handling

- Input validation loops on `y/n` and `1/2` prompts; anything else reprompts with a
  short hint. Ctrl-C / EOF exits cleanly.
- `solutions.json` missing or unparsable тЖТ warn once, fall back to live solving.
- Deck with < 4 cards remaining triggers the reshuffle prompt (13 hands per deck,
  so it lands exactly at 0 for 1 and 2 decks).

## Testing

Vitest, all non-interactive:

- **rational:** arithmetic identities, normalization, div-by-zero тЖТ null, equality.
- **cards:** value mapping A/J/Q/K edges, formatting.
- **deck:** 52/104 composition (13 ranks ├Ч 4 suits ├Ч decks), shuffle keeps multiset,
  draws deplete without replacement, deterministic under seeded RNG.
- **solver:**
  - solvable: `1,2,3,4`; `3,3,8,8` тЖТ count тЙе 1 and expression evaluates to exactly 24;
    `13,9,11,5`.
  - NO RESULT: `1,1,1,1`; `13,13,13,13`; `1,1,1,13`.
  - every returned expression is re-parsed/evaluated in the test to equal 24
    (evaluator lives in test helpers, not shipped code).
  - count sanity: dedup collapses commutative/associative variants (spot-check a
    hand with a known small solution set).
- **table:** precompute output for a few keys matches live solve; fallback path when
  file absent.
- **game:** scripted I/O runs: full happy path, invalid inputs reprompt, NO RESULT
  hand, decline at start, exhaustion тЖТ reshuffle both branches, summary correctness.
